# Android 中文输入与高刷新率补丁安装说明

如果你的游戏要发布安卓版，这个补丁是必装项。没有它时，部分安卓中文输入法可能出现候选词可见但提交为空、语音输入丢字，或弹幕输入框键盘反复弹出/收起的问题。

分发包根目录提供：

- `SDLActivity.java.mod`：Android 中文输入与高刷新率补丁参考文件。

## 目标文件

通常只需要修改 Ren'Py SDK 的 prototype 文件：

```text
Ren'Py SDK/rapt/prototype/renpyandroid/src/main/java/org/libsdl/app/SDLActivity.java
```

一般不用修改 `rapt/project`。修改 prototype 后，请在 Ren'Py Launcher 里对目标项目执行一次“清理安卓临时文件”，让项目里的 Android 临时文件重新生成，然后重新构建安卓包。

修改前一定要备份原始 `SDLActivity.java`，例如复制成 `SDLActivity.java.bak`。

## Ren'Py 8.5.0

如果你的 Ren'Py SDK 是 8.5.0：

1. 备份目标文件 `SDLActivity.java`。
2. 将分发包根目录的 `SDLActivity.java.mod` 复制到目标文件所在目录。
3. 把复制过去的文件改名为 `SDLActivity.java`，覆盖原文件。
4. 在 Ren'Py Launcher 里执行一次“清理安卓临时文件”。
5. 重新构建 Android 包，并用真机测试中文输入。

## 不是 Ren'Py 8.5.0 时

如果你的 Ren'Py 版本不是 8.5.0，不建议普通用户手动逐段合并 Java 代码，也不要直接整文件覆盖。

建议把下面文件一起交给 AI 或熟悉 Android 的开发者：

- 分发包根目录的 `SDLActivity.java.mod`（修改后完整参考文件），和 `android-patch/SDLActivity.java.patch`（unified diff 补丁，推荐）
- 本md文档
- 你的 Ren'Py SDK 原始文件：`rapt/prototype/renpyandroid/src/main/java/org/libsdl/app/SDLActivity.java`

给 AI 的要求可以这样写：

```text
请把 SDLActivity.java.mod 中的 Android 中文输入修复和高刷新率补丁，合并到我提供的原始 SDLActivity.java 中。

要求：
1. 保留我原始 SDLActivity.java 中与当前 Ren'Py 版本相关的代码。
2. 合并中文输入修复，包括 DummyEdit、SDLInputConnection、输入状态重置、onCreateInputConnection 等相关逻辑。
3. 合并高刷新率 preferredRefreshRate 代码。
4. 返回修改后的完整 SDLActivity.java。
5. 不要修改无关文件。
```

AI 合并后，把修改后的 `SDLActivity.java` 放回原目标路径，然后在 Ren'Py Launcher 里执行”清理安卓临时文件”，重新构建 Android 包，并用真机测试中文输入。

如果 patch 文件或 .mod 单独合并不顺利，还可以把三份文件一起交给 AI：SDLActivity.java.bak（8.5.0 原版）、SDLActivity.java.mod（8.5.0 修改版）和你的 Ren'Py 版本原版文件，让 AI 对比着来合并，上下文更完整，成功率更高。

## 测试方式

1. 构建 Android 包并安装到真机。
2. 打开弹幕输入框。
3. 使用中文输入法输入中文并发送弹幕。
4. 确认候选词能正常提交、语音输入不丢字、键盘不会反复弹出/收起。

升级 Ren'Py SDK 后，必须重新核对并重新应用本补丁。
